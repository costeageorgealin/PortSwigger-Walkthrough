<h1> 04. User ID controlled by request parameter, with unpredictable user IDs </h1> </br>

This lab has a horizontal privilege escalation vulnerability on the user account page, but identifies users with GUIDs.
To solve the lab, find the GUID for carlos, then submit his API key as the solution.
You can log in to your own account using the following credentials: wiener:peter </br>

<img width="1572" height="1355" alt="image" src="https://github.com/user-attachments/assets/7bdee457-8e63-40a4-b12e-e7f3a03608d1" /></br>

I began in the 'My Account' section, logging in with the provided credentials. I noticed that my user (wiener) was assigned a specific GUID/UserId, which the application used to identify session data. </br>

<img width="1162" height="1003" alt="image" src="https://github.com/user-attachments/assets/c725aec4-cca0-4964-a5e6-4de44739ce6e" /> </br>

By navigating to a blog post by Carlos and clicking his profile, I was able to retrieve his unique UserId from the URL. I then returned to the 'My Account' page and intercepted the request, replacing my UserId with Carlos's. 

<img width="1335" height="1390" alt="image" src="https://github.com/user-attachments/assets/1ba7d085-e085-4a69-afd8-af2c0e4aef3e" />

This caused the server to return Carlos’s API Key, which I then submitted to complete the lab."
